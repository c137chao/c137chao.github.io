2.Backtrace (moderate)

task2 整体的思路和实现都很简单，不断的寻找调用栈的fp，打印出返回地址，问题只有一个，，循环结束的条件是什么？

结合官方的演示和debug，正确应该是追溯三个函数 观察右边的栈帧fp（sys_sleep<-syscall<-usertrap），应该在fp = 0x3fffffa000时结束循环。

0x3fffffa000和kstack刚好相距一个页面，再依据上一个页表实验，进程栈是由高地址像低地址增长，所以，此时的fp指向的是内核栈的栈顶，同时就是backtrace所在页面的页表起始位置，usertrap再返回就要返回用户空间了，这个返回地址是不打印的。

void backtrace(void){
  // uint64 kstackpage = (uint64)myproc()->kstack;
  printf("backtrace:\n");
  uint64 fp = r_fp();
  uint64 board = PGROUNDUP(fp);
  while(fp < board){ // if over current process kernel stack
    printf("%p\n", *(uint64*)(fp - 8));
    fp = *(uint64*)(fp - 16);
  }
  return;
}

struct proc {
  // ......
  int interval;                // the alarm interval，init value = 0
  int lastcall;                // how many ticks have passed since the last 
                               // call, init value = 0
  void (*handler)();           // pointer of handler function

  // ......
};

完成sigalarm和sigreturn，sigreturn此时不做任何事情直接返回0

uint64
sys_sigalarm(void){
  struct proc * p = myproc();
  int ticks;  
  void (*fp)();
  if(argint(0, &ticks) < 0 || argaddr(1, (uint64*)&fp) < 0) 
    return -1;
  p->interval = ticks;
  p->handler  = fp;
  return 0;
}

uint64
sys_sigreturn(void){
  return 0;
}

修改usertrap，使他能够处理ticks，注意handler是一个用户空间的函数，这里不能直接执行，应该设置保存的用户pc寄存器，使handler会在下一条指令开始执行.

void
usertrap(void)
{
   // ......
  if(which_dev == 2){
    yield();    // 如果进程结束了便不应该再去执行下面的内容
    if(p->interval ){
      p->lastcall++;
      if(p->lastcall == p->interval){
        p->lastcall = 0;
        // p->handler();  // panic, don't excute handle funtion directly
        p->trapframe->epc = (uint64)p->handler; // set pc register to excute func
      }
  }
  usertrapret();
}

（2）通过test1, test2

执行handler之后，寄存器的值可能会被改变，所以之前的trapframe(进入内核之前用户空间寄存器的上下文)需要保存下来，于是在struct proc新增一个字段struct trapframe* tempframe，保存下用户寄存器的上下文，以便handler结束以后能正确恢复寄存器，使程序还能正常运行，所以只需要在usertrap里将trapframe存储的内存拷贝到tempframe，然后在sigreturn中恢复trapframe即可。

void
usertrap(void)
{
  // ...
  // give up the CPU if this is a timer interrupt.
  if(which_dev == 2){
    yield();
    if(p->interval ){
      p->lastcall++;
      if( p->lastcall == p->interval){
        memmove((void*)p->tempframe, (const void*)p->trapframe, sizeof(struct trapframe));  
        p->lastcall = 0;
        p->trapframe->epc = (uint64)p->handler;
      }
    }
  }
  usertrapret();
}

uint64
sys_sigreturn(void){
  struct proc * p = myproc();
  memmove((void*)p->trapframe, (const void*)p->tempframe, sizeof(struct trapframe)); 
  return 0;
}

注意还要仿照trapframe再allocproc和freeproc对tempframe取申请和释放内存

至此test1可以正常通过，但test2搞不定，因为还存在两个问题， (1)运行handler的期间不该再继续触发handler; (2)handler里再调用sigalarm应该是无效的
